# 15. AI 에이전트 — 상품 리뷰 분석 Agent

> **한 줄 소개(문제 중심)**: 동작만 하는 시제품 Agent를 실서비스 수준으로 끌어올리기 위해, 상품 리뷰 분석 Agent에 모니터링·고도화·대시보드를 더한 운영형 프로젝트.

### 📌 프로젝트 개요 (강의 템플릿)
| 항목 | 내용 |
|------|------|
| 문제 배경 | Agent는 동작뿐 아니라 운영 관측·재현성·결과 가시화가 필요 |
| 해결 목표 | LangSmith 모니터링 + Agent 고도화 + streamlit 대시보드 |
| 기간 / 형태 | KT 에이블스쿨 미니프로젝트 3차 |
| 역할·기여 | Agent 구현, 역할 정의서·워크플로우 명세서 설계 |
| 기술 스택 | Python, LLM Agent, LangSmith, streamlit |
| 기술 선택 이유 | **LangSmith**로 트레이싱·운영 가시성 확보 / **streamlit**로 분석 결과 대시보드를 신속 구축 |

### 🧩 문제 → 영향 → 해결 → 결과
- **문제**: Agent 운영 품질·재현성 확보와 결과의 가시화
- **영향**: 관측이 없으면 디버깅·개선이 어렵고 신뢰도 저하
- **해결**: LangSmith 트레이싱, 워크플로우 V2.0 고도화, streamlit 대시보드, 역할/워크플로우 설계 문서 작성
- **결과**: 모니터링·대시보드를 갖춘 운영형 리뷰 분석 Agent + 설계 문서

### 💡 배운 점 · 향후 개선
- **배운 점**: 운영(모니터링) 관점 설계, 설계 문서 기반 Agent 구성
- **향후 개선**: 평가 지표 자동화, 프롬프트/모델 A/B 비교

| 산출물 | 노트북, 발표자료, 설계 문서(역할 정의서·워크플로우 명세서), 데이터 |

## 핵심 구성

| 미션 | 내용 |
|------|------|
| 미션③ 모니터링 | **LangSmith** 기반 Agent 운영 모니터링 시스템 구축 |
| 미션④ 고도화 | State·LLM·Agent 노드 구성 개선, 워크플로우 V2.0 |
| 미션⑤ 대시보드 | **streamlit** 기반 분석 결과 대시보드 제작 |

## 설계 산출물

- **Agent 역할 정의서**(`Step2_Agent 역할 정의서_양식.xlsx`): 각 Agent 노드의 역할·입출력 정의
- **Workflow 명세서**(`Step2_Workflow 명세서 외 기타 설계문서들_양식.xlsx`): 전체 워크플로우·분기·상태 설계
- 분석 데이터: `data_gender.csv`

## 기술 요소

- LLM Agent 파이프라인(State 관리, 멀티 노드)
- **LangSmith** 트레이싱·모니터링으로 운영 관점 검증
- **streamlit** 대시보드로 결과 시각화
- 설계 문서 기반 체계적 Agent 설계
